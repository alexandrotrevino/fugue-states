# C2 — Fugue States Remote Control Protocol

The Pi runs autonomously (systemd unit `fugue-states.service`). C2 is the
OSC vocabulary that lets a controller — initially PD/PlugData, eventually
a dedicated app — drive the Pi at performance time without SSH or
`systemctl`. The data plane (`/<MAC>/<sensor>`) is unchanged; C2 lives in
disjoint address spaces.

## Scope

What C2 covers:
- Stream control (start/stop, per-device or broadcast)
- Status query and proactive state push
- Reconfiguration of sensor and network settings (when not streaming)
- Position-tracker recalibration
- Heartbeat + error reporting

What C2 does **not** cover:
- Code deployment, OS-level restart, log retrieval — stays via SSH /
  `systemctl` / `journalctl`.
- Service discovery — Pi IP goes in the controller's static config for
  v1. mDNS is a Phase-2 layer above C2.
- Multi-controller arbitration. One controller at a time; replies always
  go to the configured target.
- Authentication beyond the shutdown token. Assumes a trusted LAN.

## Transport

OSC over UDP. Pi listens on `0.0.0.0:8001` (existing
`ControlledOSCConnection`). Replies and async events go to the OSC target
configured in `fs_config.local.json` (same target as data). The
controller is responsible for being reachable at that address.

## Address layout

```
/<MAC>/<sensor>             — data plane, unchanged
/<MAC>/__advertise__        — capabilities, sent once after connect

/cmd/<verb>                 — controller → Pi
/state/<scope>/<key>        — Pi → controller (snapshots, change events,
                              heartbeat)
/error/<code>               — Pi → controller (async failures)
```

`<MAC>` is the colon-separated lowercase form already used on the data
plane (e.g. `ee:21:4e:39:12:34`).

## Argument conventions

- All commands and state messages use **flat scalar args**: ints,
  floats, strings. No bundles, no nested lists. PD's `[route]` and
  `[unpack]` consume these directly.
- Booleans are encoded as int `0`/`1`.
- Optional MAC args: empty string `""` means "broadcast / all devices".
- Address paths are stable; arity per address never varies across
  versions of the same address.

## Shutdown token

`/cmd/shutdown` requires a token to prevent stale or accidental shutdown
from killing the Pi.

- **Generation**: at process start, Pi generates a 6-char hex token via
  `os.urandom(3).hex()`. Lives in memory only; new PID → new token.
- **Distribution**: every `/state/heartbeat` carries the token as its
  second argument. Any controller observing the Pi for ≥ 2 s holds the
  current token.
- **Validation**: on `/cmd/shutdown <provided>`, Pi compares against the
  live token. Match → accept and stop. Mismatch or missing →
  `/error/bad-token <reason>` and ignore.

This defends against:
- Stale `.pd` patches with shutdown bangs from prior sessions.
- Misrouted shutdown messages from other controllers on the LAN.
- Replay from a previous Pi session.

It does **not** defend against a malicious actor sniffing UDP on the
LAN. That threat model is out of scope (would require TLS / TCP control
channel).

## Commands (controller → Pi)

| Address | Args | Behaviour |
|---|---|---|
| `/cmd/status` | — | Triggers a state-snapshot reply: one `/state/<MAC>/snapshot` per device plus `/state/global/snapshot`. |
| `/cmd/start` | `[mac:str]` | Start streaming. Empty MAC = all devices. Idempotent — already-streaming devices are skipped silently. Each device that transitions emits `/state/<mac>/streaming 1`. |
| `/cmd/stop` | `[mac:str]` | Stop streaming. Empty MAC = all devices. Idempotent. Each device that transitions emits `/state/<mac>/streaming 0`. |
| `/cmd/configure/sensor` | `<mac:str> <sensor:str> <key:str> <value:str-or-num>` | Set one sensor knob (e.g. `acc odr 25`, `"Sensor Fusion" outputs "linear_acc,quat,corrected_gyro"`). Rejected with `/error/configure-rejected streaming` while any device is streaming. Persisted to `fs_config.local.json`. Acks `/state/configured sensor <mac>/<sensor>/<key>`. |
| `/cmd/configure/network` | `<ip:str> <port:int>` | Change OSC data target. Persisted to `fs_config.local.json`. Acks `/state/configured network <ip>:<port>`. The new target receives subsequent traffic — including the ack itself. |
| `/cmd/calibrate` | `[mac:str]` | Force position-tracker recalibration. Re-enters cold-start (yellow LED, 5 s stationary). No-op if `--position-track` wasn't enabled at process start. |
| `/cmd/shutdown` | `<token:str>` | Stop the OSC server and exit cleanly. Token must match current heartbeat token. |

## State (Pi → controller)

| Address | Args | Emitted |
|---|---|---|
| `/state/heartbeat` | `<uptime_s:float> <token:str>` | Every 2 s. |
| `/state/<mac>/connected` | `<value:int>` (0 or 1) | On change. |
| `/state/<mac>/streaming` | `<value:int>` (0 or 1) | On change. |
| `/state/<mac>/snapshot` | `<connected:int> <streaming:int> <recording:int> <error_count:int>` | One per device, in reply to `/cmd/status`. |
| `/state/global/snapshot` | `<num_devices:int> <position_track:int> <recording:int> <uptime_s:float>` | In reply to `/cmd/status`. |
| `/state/recording` | `<value:int> [path:str]` | On record start (`1 <path>`) or stop (`0`). |
| `/state/configured` | `<scope:str> <key:str>` | After a successful `/cmd/configure/*`. `scope` is `sensor` or `network`. |
| `/state/<mac>/calibrating` | `<value:int>` | Position tracker entering (`1`) or finishing (`0`) cold-start calibration. |

## Errors (Pi → controller)

| Address | Args | Meaning |
|---|---|---|
| `/error/already-streaming` | `<mac:str>` | `/cmd/start` issued, device was already streaming. Informational; the device stays streaming. |
| `/error/not-streaming` | `<mac:str>` | `/cmd/stop` issued, device wasn't streaming. Informational. |
| `/error/not-connected` | `<mac:str>` | Command targeting a device that isn't connected to BLE. |
| `/error/configure-rejected` | `<reason:str>` | `streaming` (devices live), `bad-key`, `bad-value`, `unknown-sensor`. |
| `/error/bad-token` | `<reason:str>` | `mismatch` or `no-token-yet`. Returned for `/cmd/shutdown` with wrong/missing token. |
| `/error/calibrate-failed` | `<mac:str> <reason:str>` | `not-enabled` (no `--position-track`), `not-streaming` (no input frames), etc. |
| `/error/ble-drop` | `<mac:str>` | Async — sensor link dropped. Watchdog will attempt recovery; `/state/<mac>/connected 0` follows. |
| `/error/stale-stream` | `<mac:str>` | Async — watchdog detected no frames for the stale threshold. Recovery in progress. |

## Example flows

**Operator brings up a session from PD.**

```
controller → Pi  /cmd/status
Pi → controller  /state/ee:21:.../snapshot  1 0 0 0
Pi → controller  /state/ff:11:.../snapshot  1 0 0 0
Pi → controller  /state/global/snapshot     2 1 0 312.4

controller → Pi  /cmd/start ""              # broadcast
Pi → controller  /state/ee:21:.../streaming 1
Pi → controller  /state/ff:11:.../streaming 1

# (heartbeats every 2s in the background)
Pi → controller  /state/heartbeat 314.1 a3f7c1
```

**Operator changes the OSC target mid-session (must stop streaming first).**

```
controller → Pi  /cmd/stop ""
Pi → controller  /state/ee:21:.../streaming 0
Pi → controller  /state/ff:11:.../streaming 0

controller → Pi  /cmd/configure/network "192.168.4.50" 12345
Pi → controller  /state/configured network 192.168.4.50:12345
                 # delivered to the NEW target
```

**Accidental shutdown bang from a stale patch.**

```
old-patch → Pi   /cmd/shutdown "0000aa"     # token from a prior session
Pi → controller  /error/bad-token mismatch
                 # Pi keeps running
```

## Versioning

C2 v1 is what's specified here. Address paths and arity per address are
stable contracts — additions are non-breaking, but renames or arg-shape
changes require a v2 spec doc and a transition plan. The dedicated app
in Phase 2 is expected to ride on v1 unchanged; richer abstractions
(service discovery, gesture-state UI, position visualizations) layer
*above* the OSC vocabulary, not by replacing it.

## Implementation notes

- The handlers live in `MetaWearState.set_OSC` (per-device). Broadcast
  commands fire on every device because every state instance maps the
  same address; per-device commands filter by matching `args[0]` against
  `self.address`.
- Reply addresses come from `self._osc_client` (the same client used for
  data) — replies inherit any `fs_config.local.json` target override.
- Heartbeat is driven from `run_fs.py`'s watchdog loop (1 Hz tick); emit
  every other tick to land on 2 s.
- `/state/configured` is the ack signal — no separate "/cmd/configure
  accepted" message. The presence of `/state/configured` *is* the ack.
